# Podcast Translation Skill Contract

这份文档定义 `podcast-translation-skill` 应该如何封装现有 Python 播客翻译工作流

## 结论

当前Agent系统 little_claw 已经具备以下能力：

- `TaskQueue` 可以持久化 agent 任务状态，包括 `assigned`、`running`、`awaiting_approval`、`completed`、`failed`。
- `TeamScheduleStore`、`TeamCronScheduler`、`TeamScheduleAdapter` 可以把定时触发转换成 team task。
- `AgentWorker` 可以执行任务、等待人工审批、把结果写回 project channel 和 `context-hub/3-projects/{project}/status.md`。
- `Podcast Curator` 模板已经把 `timeout_minutes` 设为 120 分钟，足够覆盖长流程外部任务的管理任务。

但当前 little_claw 不应该直接承载 1 小时音频翻译进程：

- 通用 `shell` 工具默认超时是 30 秒；虽然可以传 `timeout_ms`，但让 Agent 等一小时会占用一个 worker 执行窗口。
- `CoordinatorLoop` 会按 agent 的 `timeout_minutes` 扫描并失败运行中的 task，适合管理任务，不适合把外部长进程当作一次 blocking tool call。
- Skill 在当前实现里是“指令和资源”，不是可直接调用的 tool。Agent 仍然通过 `shell`、MCP tool 或未来自定义 tool 执行命令。

因此推荐架构是：

```text
Podcast Curator
  -> podcast-translation-skill instructions
  -> short CLI/API call: start translation
  -> returns job_id immediately
  -> record job_id in project state
  -> scheduled status check task polls job status
  -> completed/error result is posted to #podcast-translation
```

你的主要工作应该放在 Python 工作流仓库：把原先交互式 CLI 改造成非交互、JSON 输出、异步 job 状态机。

## Required CLI Shape

所有命令必须满足：

- 非交互：不能出现菜单选择、`input()`、确认提示。
- JSON 输出：成功和失败都输出单个 JSON object。
- 稳定退出码：成功为 `0`；参数错误、找不到资源、认证错误、任务失败分别使用非零退出码。
- 幂等：重复查询状态、重复读取 RSS 不应产生副作用。
- 可被 `uv` 运行：示例统一使用 `uv run podcast_tool ...`，实际模块名可以调整，但 Skill 文档中必须写死真实命令。

### RSS Discovery

根据关键词或播客名称查找 RSS。

```bash
uv run podcast_tool rss find --query "Acquired podcast" --json
```

输出：

```json
{
  "ok": true,
  "query": "Acquired podcast",
  "feeds": [
    {
      "title": "Acquired",
      "publisher": "Ben Gilbert and David Rosenthal",
      "rss_url": "https://example.com/feed.xml",
      "website_url": "https://example.com",
      "description": "Long-form business history podcast.",
      "language": "en",
      "confidence": 0.96
    }
  ],
  "error": null
}
```

要求：

- `feeds` 按可信度从高到低排序。
- `confidence` 范围为 `0` 到 `1`。
- 如果没有找到，返回 `ok: true`、`feeds: []`。

### Feed Episodes

读取 RSS 中的 episode 列表。

```bash
uv run podcast_tool episodes list --rss-url "https://example.com/feed.xml" --limit 10 --json
```

输出：

```json
{
  "ok": true,
  "rss_url": "https://example.com/feed.xml",
  "feed": {
    "title": "Acquired",
    "publisher": "Ben Gilbert and David Rosenthal",
    "language": "en"
  },
  "episodes": [
    {
      "episode_id": "stable-id-or-guid",
      "title": "Episode title",
      "published_at": "2026-05-01T10:00:00Z",
      "audio_url": "https://example.com/audio.mp3",
      "page_url": "https://example.com/episode",
      "duration_seconds": 5400,
      "shownotes_original": "Original show notes text.",
      "shownotes_zh": "Optional Chinese summary if generated by this command."
    }
  ],
  "error": null
}
```

要求：

- `episode_id` 必须稳定。优先用 RSS guid；没有 guid 时用 `audio_url` 或 `page_url` 的 hash。
- `published_at` 使用 ISO 8601。
- `shownotes_original` 可以是纯文本或保守清洗后的 HTML 文本，但不要返回整页噪声。

### Start Translation

启动英文音频到中文音频的异步翻译流程。

```bash
uv run podcast_tool translate start \
  --rss-url "https://example.com/feed.xml" \
  --episode-id "stable-id-or-guid" \
  --target-lang "zh-CN" \
  --voice-clone true \
  --json
```

也可以支持直接传 episode 元数据：

```bash
uv run podcast_tool translate start \
  --audio-url "https://example.com/audio.mp3" \
  --title "Episode title" \
  --page-url "https://example.com/episode" \
  --target-lang "zh-CN" \
  --voice-clone true \
  --json
```

输出必须立刻返回，不等待完整翻译：

```json
{
  "ok": true,
  "job": {
    "job_id": "podcast_20260503_9f3a2c",
    "status": "queued",
    "stage": "queued",
    "progress": 0,
    "created_at": "2026-05-03T08:30:00Z",
    "updated_at": "2026-05-03T08:30:00Z",
    "estimated_minutes": 60,
    "episode": {
      "rss_url": "https://example.com/feed.xml",
      "episode_id": "stable-id-or-guid",
      "title": "Episode title",
      "audio_url": "https://example.com/audio.mp3",
      "page_url": "https://example.com/episode",
      "published_at": "2026-05-01T10:00:00Z"
    },
    "artifacts": {}
  },
  "error": null
}
```

要求：

- `job_id` 是后续所有查询的唯一句柄。
- `start` 成功后，后台翻译进程必须独立于调用 shell 存活。
- 如果相同 episode 已经有未完成 job，默认返回已有 job，除非传 `--force`。
- 必须把 job 元数据持久化到 Python 工作流自己的状态目录或数据库。

```bash
uv run podcast_tool translate status --job-id "podcast_20260503_9f3a2c" --json
```

输出：

```json

    "stage": "tts",
```

失败时：

```json
{
  "ok": false,
  "job": {
    "job_id": "podcast_20260503_9f3a2c",
    "status": "failed",
    "stage": "tts",
    "progress": 0.68,

  },
  "error": {
    "code": "TTS_PROVIDER_ERROR",
    "message": "Aliyun TTS request failed: ...",
    "retryable": true
  }

```

Allowed `status` values:

- `queued`
- `running`
- `completed`
- `failed`
- `cancelled`

Recommended `stage` values:

- `queued`
- `download`
- `voiceprint`
- `stt`
- `translate`
- `tts`
- `shownotes`
- `packaging`
- `done`
- `failed`
- `cancelled`

### List Active Jobs

用于 Podcast Curator 的定时巡检。

```bash
uv run podcast_tool translate list --status active --json
```

输出：

```json
{
  "ok": true,
  "jobs": [
    {
      "job_id": "podcast_20260503_9f3a2c",
      "status": "running",
      "stage": "tts",
      "progress": 0.72,
      "updated_at": "2026-05-03T09:10:00Z",
      "episode": {
        "title": "Episode title",
        "audio_url": "https://example.com/audio.mp3"
      }
    }
  ],
  "error": null
}
```

`--status active` 应包含 `queued` 和 `running`。

## Skill Directory

把 Skill 放在任一目录：

```text
.little_claw/skills/podcast-translation-skill/SKILL.md
~/.little_claw/skills/podcast-translation-skill/SKILL.md
```

当前 `SkillLoader` 会扫描这两个位置。项目级 `.little_claw/skills` 优先级高于全局 `~/.little_claw/skills`。

推荐 `SKILL.md` frontmatter：

```markdown
---
name: podcast-translation-skill
description: Use this skill when discovering podcast RSS feeds, listing podcast episodes, translating English podcast audio into Chinese audio, cloning or preserving speaker voice, checking asynchronous translation job status, or delivering translated podcast artifacts.
metadata:
  openclaw:
    requires:
      bins:
        - uv
      config:
        - ~/.little_claw/podcast_tool/config.json
---
```

`SKILL.md` body 应该只保留操作规则，不要写长篇背景。核心内容：

```markdown
# Podcast Translation Skill

Use the Python CLI through `shell`. All commands must include `--json`.

## RSS
- Find RSS: `cd /path/to/python/repo && uv run podcast_tool rss find --query "<name or keywords>" --json`
- List episodes: `cd /path/to/python/repo && uv run podcast_tool episodes list --rss-url "<rss_url>" --limit 10 --json`

## Translation
- Start translation only after explicit human approval.
- Start command: `cd /path/to/python/repo && uv run podcast_tool translate start --rss-url "<rss_url>" --episode-id "<episode_id>" --target-lang "zh-CN" --voice-clone true --json`
- Never wait for the full translation in a single shell call.
- After start returns, record `job_id` in `context-hub/3-projects/podcast-translation/active-jobs.md` or `active-jobs.json`.

## Status
- Poll status: `cd /path/to/python/repo && uv run podcast_tool translate status --job-id "<job_id>" --json`
- List active jobs: `cd /path/to/python/repo && uv run podcast_tool translate list --status active --json`
- When completed, deliver `audio_zh`, `shownotes_zh`, and a short Chinese summary.
- When failed, report `error.code`, `error.message`, and whether it is retryable.
```

## Podcast Curator Workflow

### Daily discovery

The existing `cron_jobs` mechanism can create a daily task for Podcast Curator. That task should:

1. Read subscribed feeds and interest memory from `context-hub/3-projects/podcast-translation/`.
2. Use `rss find` only when an RSS URL is missing.
3. Use `episodes list` for known feeds.
4. Translate titles and show notes into concise Chinese.
5. Recommend 1 to 3 episodes.
6. Ask the user which episode to translate.
7. Stop. Do not start translation without explicit approval.

### User approves an episode

After approval:

1. Call `translate start`.
2. Parse `job.job_id`.
3. Write a record to project state:

```json
{
  "job_id": "podcast_20260503_9f3a2c",
  "status": "queued",
  "episode_title": "Episode title",
  "rss_url": "https://example.com/feed.xml",
  "episode_id": "stable-id-or-guid",
  "started_at": "2026-05-03T08:30:00Z",
  "last_checked_at": null
}
```

4. Reply to the user that the translation has been dispatched and will be reported later.

### Status polling

Use a separate scheduled task for status checks, for example every 10 minutes:

```yaml
cron_jobs:
  - key: podcast-translation-status-check
    name: Podcast Translation Status Check
    cron: "*/10 * * * *"
    prompt: "Check active podcast translation jobs in the podcast-translation project. Use podcast-translation-skill status commands. Report only completed or failed jobs; otherwise update project state quietly."
    project: podcast-translation
    tags:
      - scheduled
      - podcast
      - translation
    priority: 0
    max_retries: 1
    enabled: true
```

The status task should:

1. Read active job ids from project state or call `translate list --status active`.
2. Query `translate status` for each active job.
3. If still `queued` or `running`, update project state and avoid noisy user messages.
4. If `completed`, post the final artifacts and remove it from active jobs.
5. If `failed` or `cancelled`, post a concise failure report and remove it from active jobs.

## Current little_claw Changes Needed

For the first working version, no core runtime change is required if the Python side implements the CLI contract above.

Recommended small follow-ups in little_claw:

- Ensure Podcast Curator has a status polling `cron_jobs` entry in `agent.yaml`.
- If `podcast-translation-skill` combines RSS discovery and translation, remove stale references to `podcast-rss-fetcher` from the agent config.
- If you rely on network search, add a real `web_search` tool or MCP server. The current built-in tool registry does not register `web_search`.
- If you need guaranteed skill injection, pin the skill in `~/.little_claw/config.json`:

```json
{
  "skills": {
    "tokenBudget": 20000,
    "pinned": ["podcast-translation-skill"],
    "entries": {}
  }
}
```

Important: `agent.yaml.skills` is currently useful metadata for the agent, but Skill instructions are loaded by `SkillManager` from `.little_claw/skills` and `~/.little_claw/skills`, then selected by retrieval or pinning.

## Validation Checklist

Before connecting to Podcast Curator:

- `rss find` returns valid JSON for a known podcast name.
- `episodes list` returns stable `episode_id` values.
- `translate start` returns within 10 seconds and includes `job.job_id`.
- The backend job continues if the shell process exits.
- `translate status` works after restarting the Python CLI process.
- `translate list --status active` returns all queued/running jobs.
- Completed jobs include absolute artifact paths.
- Failed jobs include `error.code`, `error.message`, and `retryable`.
- No command waits for terminal input.
- All commands work from a clean shell with only `cd /path/to/python/repo && uv run ...`.
